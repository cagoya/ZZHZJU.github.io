# 数据库系统

## 评分标准

* 平时作业 + lab + minisql + 随堂小测：50%
* 期末考试：50%

## 个人感受

数据库平时作业抄抄答案差不多了，期末考试前再集中复习理论。数据库有 4 个 lab，前三个都是纯 SQL 的，不是很难，不建议用学校给的服务器，因为学校服务器的数据库版本很低，有些语法直接不支持，而且可能会被其他人删表，建议在本地完成，最后一个 lab 是图书管理系统，前后端框架都已经给出，后端看看 JDBC 写下很快的，因为全是差不多的函数，后端单元测试都是写好的，直接面向测试点编程，前端因为现在有框架和 AI，写起来很快；至于大程 minisql，现在有 AI，难度也不是那么大，遇到实在有疑问的地方去网上找下参考，minisql 建议 7 个部分都做，因为minisql对于理论知识的理解还是有比较大的帮助的，可以尽早开始，尽量在倒数第二周就去验收。

期末考试今年改成了只有客观题，也就是选择 + 判断 + 填空，感觉整体难度是降低了，但是对知识的考察还是很细，考前建议综合不同老师 PPT 和教材仔细复习，今年不写复杂的 SQL 了，感觉是一大进步。历年卷还是建议尽量多做，因为虽然题型变了，但是考察点几乎没变，历年卷的答案参考一下就行，因为有些答案很扯，尤其是查询处理和查询优化，很多答案简直莫名其妙。
