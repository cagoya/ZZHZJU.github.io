# 事务

## 简介

事务 Transaction 主要要解决两个问题：

1. Failures of various kinds, such as hardware failure and system crashes
2. Concurrent execution of multiple transactions

事务需要满足ACID：

* Atomicity:原子性，要么事务的所有操作都完成了，要不就都没完成
* Consistency:一致性，独立并发执行的操作能够保持数据的一致，比如满足主键约束和外键约束
* Isolation:隔离性，主要是为了能够并发执行，不同事务之间不会相互影响 ，事务之间相互不可见
* Durability:持久性，和恢复有关，如果一个事务提交了，对数据库的修改就必须永久保持

一个事务的状态有：

1. Active 正常执行
2. Partially commit 最后一条语句已经执行
3. Failed 发现无法正常执行，比如发现违背了外键约束
4. Aborted 失败但是恢复了
5. Committed   完全成功完成

## 并发控制

### 并发可能异常

并发如果不加以控制会遇到以下这四类异常：

1. Lost Update 丢失修改，例子是两个事物同时写-1的结果，只反映一次修改的结果
2. Dirty Read 读脏数据，例子是一个事务回滚了，但是另一个事务读了它修改过的数据
3. Unrepeatable Read，不能反复读，例子是一个事务修改了数据导致另一个事务读到的前后不一致，这里是针对一个已经存在的数据，这个问题比较好解决，比如加个锁
4. Phantom Problem，幽灵（幻影）问题，例子是一个事务进行了查询，但是另一个进行了插入或者删除，这里是针对一张表的 tuple，这个很不好解决，解决的代价较高，最后会讲

事务的并发执行会产生很多问题，但是还是要并行而不是串行，主要是因为：

1. 增加CPU和磁盘利用率
2. 减少平均响应时间

并发与并行是两个不同的概念：

* 并行 (Parallelism)，同时执行多个任务或者操作，通常涉及到多个处理单元或多台计算机同时执行不同的任务，是**真正的同时执行**，例子是在多核处理器上同时执行多个线程，或者将计算任务分发给多台服务器同时处理它们（即分布式系统）
* 并发 (Concurrency)，并发是指多个任务在同一时间段内执行，它们可能交替执行，但**不一定在完全相同的时刻**，更强调任务切换和调度，模拟“同时进行”，比如一个程序先用磁盘再用CPU，另一个先用CPU再用磁盘，典型场景式网络服务/异步编程

### 可串行化

调度 Schedule 是一系列并发事务的所有操作的实际执行顺序，这个过程中每个事务里面的执行顺序一定是保持不变的，成功执行的事务会以 commit 结束，失败执行的事务会以 abort 结束。

串行调度就是一个事务先做另一个后做，**串行调度一定是满足隔离性的**，因为它们在时间上面没有交叉，并发调度就未必了，访问不同的数据是不冲突的，同时读也是不冲突的，但是对同一个数据写和写，写和读都是**冲突的**。

这里有一个重要概念是可串行化 Serializability，一个交叉调度如果是等价于一个串行调度的，那么就是一个可串行调度，可串行性从不同角度看有不同的理解，这门课只考虑冲突可串行化。如果我们能够通过交换一系列不冲突的操作把`S`转化成`S'`，那么它们就是冲突等价的。

前驱图(precedence Graph)是一个有向图，顶点是事务的名称，边存在于冲突的事务之间，如果事务$T_i$的一个操作与$T_j$的一个操作冲突，且在调度中$T_i$的操作先于$T_j$的操作，则画一条从$T_i$到$T_j$的边，边上可以写访问的数据。一个调度是冲突可串的当且仅当它的前驱图是没有环的(acyclic)，判断有向图是否有环可以用一个DFS，未访问的顶点都标记为0，递归栈入栈时写为1，出栈时写为2，如果DFS过程中访问到了标记为1的节点，说明有环，时间复杂度为$O(V+E) $。如果一个前驱图是无环的，那么它的可串行化执行顺序可以通过拓扑排序(topological sorting)获得。

### 可恢复

可恢复要求如果一个事务$T_j$读了另一个事务$T_i$修改过的数据，那么$T_j $的提交必须晚于$T_i$，因为$T_i$有可能回滚导致$T_j $读取的$T_i $修改的数据作废，这种情况下$T_i $回滚，$T_j $也会跟着回滚，这个叫做级联回滚(cascading rollback)，过多的级联回滚会导致大量事务重做，这个会有很大的开销。因此，为了无级联的调度(cascadeless schedule)，要求$T_j $在读取被$T_i $修改过的数据时$T_i $已经提交，这样$T_i $就没有机会回滚了，显然，无级联调度的要求是比可恢复调度更强的。

### 隔离性

数据库一般有四种隔离级别：

- 可串行 serializable：默认
- 可重复读 repeatable read：只有提交的记录会被读，一个事物重复读同一条记录必须返回相同的结果。
- 读已提交 read committed：只读已提交的记录，但是连续的读结果可以不一样。
- 读未提交 read uncommitted：未提交的数据也可以读。
